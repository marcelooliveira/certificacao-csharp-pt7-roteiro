Sintaxe de Consulta Linq
==========================

Você é chamado por um colega desenvolvedor para ajudá-lo com uma consulta Linq. O seu colega acha que a consulta que ele criou está correta, mas que mesmo assim o programa não está compilando. Você pede para ver a consulta que ele criou, você encontra:

```
List<Cliente> clientes = new List<Cliente>()
                {
                    new Cliente { Id = 1001, Nome = "José Raimundo Bernardino" },
                    new Cliente { Id = 1002, Nome = "Alexandre Marciano da Luz" },
                    new Cliente { Id = 1003, Nome = "Milton Guilherme da Silva" }
                };

                var queryClientes =
                select cli
                from cli in clientes;
```

Qual a causa provável para o problema?

> **Mão na massa**: se quiser praticar online, acesse: [Sintaxe de Consulta Linq](https://dotnetfiddle.net/hy6v5t)

--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

-   Alternativa

A consulta não está contida entre parênteses.

Uma definição de consulta não precisa estar contida entre parênteses.

-   Alternativa

O nome da variável `cli` é diferente do nome da coleção `clientes`.

O nome da variável cli independe do nome da coleção.

-   Alternativa

A cláusula select não declara a lista de propriedades do cliente a serem trazidas pela consulta.

Uma consulta Linq não precisa descrever necessariamente as propriedades da coleção clientes. A variável cli da cláusula select já permite acessar essas propriedades.

-   Alternativa

A tabela `clientes` não está no banco de dados, e precisa ser criada antes da consulta.

A coleção `clientes` não é uma tabela, e sim uma coleção em memória.

-   Alternativa

A consulta Linq implementada está começando pela cláusula *select*.

CORRETO. Uma expressão Linq precisa começar pela cláusula from, e não pela cláusula select:

```
    var queryClientes =
                            from cli in clientes
                            select cli;
```
